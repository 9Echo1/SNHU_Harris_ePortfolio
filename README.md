# SNHU_CS465_FULL-STACK_DEV1

Architecture
  In my full stack project, I worked with Express HTML, JavaScript, and a single-page application (SPA) to create the Travlr Getaways web application. I used Express HTML, which allows for dynamic server-side rendering, while JavaScript handles client-side interactivity. SPA, on the other hand, offers a smoother user experience by loading content dynamically without full page reloads. I used MongoDB for the backend because of its flexibility with unstructured data and ability to scale horizontally, making it ideal for applications that may evolve or handle large amounts of data.


Functionality
  For its functionality, we used JSON.  JSON is a data format that differs from JavaScript as it's used to transfer data between frontend and backend systems. It ties the two together by ensuring data is easily exchanged and displayed. In the project, I refactored code by breaking down large components and making UI elements reusable, improving maintainability and efficiency.


Testing
  Postman was instrumental in API testing of endpoints.  Testing API endpoints involves methods like GET, POST, PUT, and DELETE, and ensures data is correctly transmitted between client and server. Security, including authentication, adds complexity, as tests need to verify that only authorized users can access specific endpoints. I tested protected routes to ensure that sensitive data remained secure.


Reflection
  This course helped me enhance my full stack development skills by familiarizing me with JavaScript frameworks, backend systems, and NoSQL databases like MongoDB. I believe this class and these skills have made me more marketable and have prepared me to contribute to building scalable web applications.
